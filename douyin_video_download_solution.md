# 抖音视频下载与转文字桌面客户端技术方案

## 1. 背景

本方案用于开发一个桌面客户端 APP。用户在客户端中输入抖音视频 URL 后，系统自动下载公开视频、提取音频、调用本地 ASR 模型识别语音内容，再接入 InsightFlow 从文字稿中生成具有启发性的开放式话题点，最终输出话题点和完整文字稿。

示例链接：

```text
https://www.douyin.com/video/7524373044106677544
```

本次已验证基础下载链路，视频已成功保存至：

```text
outputs/7524373044106677544.mp4
```

## 2. 目标

- 支持通过抖音视频 URL 下载公开视频。
- 输出标准 MP4 视频文件。
- 下载后校验文件是否为有效媒体文件。
- 在桌面客户端内输入视频 URL 后，一键输出视频语音转写文本。
- 使用 `Qwen/Qwen3-ASR-0.6B` 作为默认中文 ASR 模型。
- 接入 InsightFlow，从文字稿中生成具有启发性的开放式话题点。
- 支持导出文字稿 `txt` / `md`，以及话题点 `json` / `md`；后续可扩展 `srt` 字幕文件。
- 保留可复用的命令流程，便于后续批量化、自动化或产品化扩展。

## 3. 技术选型

### 3.1 下载工具

使用 `yt-dlp` 作为核心下载工具。

选择原因：

- 支持抖音等主流短视频平台解析。
- 可自动提取页面中的视频资源地址。
- 支持指定输出路径、文件名模板、格式选择、cookies 等参数。
- 命令行方式易于集成到脚本、任务调度或后端服务中。

本次环境中的 `yt-dlp` 版本：

```text
2026.03.17
```

### 3.2 媒体校验工具

使用 `ffprobe` 读取视频容器、音视频流、分辨率、时长和文件大小。

选择原因：

- 是 FFmpeg 工具链的一部分，稳定可靠。
- 可用于确认下载结果不是空文件、错误页面或异常响应。
- 输出 JSON，方便后续程序化处理。

### 3.3 音频预处理工具

使用 `ffmpeg` 从视频中提取音频，并统一转为 ASR 友好的格式。

推荐音频格式：

```text
WAV / mono / 16000 Hz / pcm_s16le
```

示例命令：

```powershell
ffmpeg -y -i "outputs/7524373044106677544.mp4" `
  -vn -ac 1 -ar 16000 -c:a pcm_s16le `
  "work/7524373044106677544.wav"
```

### 3.4 ASR 语音识别模型

默认选择：

```text
Qwen/Qwen3-ASR-0.6B
```

选择原因：

- 相比 Whisper tiny/base/small，更适合中文、粤语和多种中文方言场景。
- 相比 `Qwen3-ASR-1.7B`，资源消耗更低，更适合桌面客户端默认内置或首次下载使用。
- 支持本地离线推理；如后续采用 vLLM backend，可继续扩展流式识别。
- 官方提供 `qwen-asr` Python 包，便于封装为本地 worker。

模型下载建议：

```powershell
pip install -U qwen-asr
pip install -U modelscope
modelscope download --model Qwen/Qwen3-ASR-0.6B --local_dir ./models/Qwen3-ASR-0.6B
```

说明：

- 中国大陆环境优先使用 ModelScope 下载模型权重。
- 如网络环境稳定，也可以从 Hugging Face 下载 `Qwen/Qwen3-ASR-0.6B`。
- 桌面客户端不建议把大模型权重直接打进安装包，建议首次运行时下载，或在设置页提供模型管理能力。

### 3.5 桌面客户端技术栈

选定 MVP 技术栈：

```text
Tauri + React + TypeScript + Python ASR Worker
```

职责划分：

- `Tauri`：桌面壳、文件权限、进程管理、应用打包。
- `React + TypeScript`：输入 URL、任务进度、文字结果、导出按钮等界面。
- `Python Worker`：调用 `yt-dlp`、`ffmpeg`、`ffprobe`、`qwen-asr`，以及复制到本项目内的 InsightFlow 话题点生成逻辑。
- `SQLite`：可选，用于保存历史任务、原始 URL、输出路径、处理状态。

### 3.6 InsightFlow 参考实现与话题点生成

InsightFlow 参考代码来源：

```text
D:\Github\InsightFlow\src\server
```

该路径仅作为研发参考和代码迁移来源，不作为桌面客户端运行期依赖。开发时应将必要代码复制或改造到当前项目目录，例如：

```text
project-root/
  worker/
    insightflow/   # 从 InsightFlow 参考代码复制和裁剪后的话题点生成模块
```

运行期禁止从 `D:\Github\InsightFlow\src\server` 跨目录 `import` 或要求用户保留该仓库路径。这样桌面客户端才能独立打包、安装和运行。

参考实现中可复用的能力：

- `POST /upload/{user_id}`：上传 Markdown 或文本文件。
- `POST /questions/generate/{user_id}/{file_id}`：异步生成启发式问题。
- `GET /file_status/{file_id}`：查询文件处理状态。
- `GET /questions/{file_id}`：获取生成的问题列表。

InsightFlow 的问题生成 prompt 已定位为“阅读思考伙伴和议题策展者”，目标是从文本中提炼开放式议题问句。对本产品而言，这些问题在界面中命名为：

```text
启发话题点
```

集成方式：

- 桌面客户端完成 ASR 后，将文字稿保存为 Markdown 文件。
- Python Worker 调用本项目内置的 InsightFlow 话题点生成模块。
- 内置 InsightFlow 模块对文字稿切块并调用 LLM 生成开放式话题点。
- 桌面客户端获取话题点列表，并与文字稿一起展示。
- 如果内置 InsightFlow 模块初始化失败，客户端进入 `部分完成` 状态，仅展示文字稿并允许稍后重试生成话题点。

## 4. 目录规划

项目目录建议包含以下主要部分：

```text
project-root/
  app/       # 桌面客户端前端和 Tauri 工程
  worker/    # Python 下载、转码、ASR 处理逻辑
    insightflow/  # 复制和裁剪后的 InsightFlow 话题点生成逻辑
  models/    # 本地模型权重缓存
  outputs/   # 用户可直接使用的最终视频和转写结果
  work/      # 中间文件、音频、调试信息、临时脚本
```

## 5. 桌面客户端产品需求

### 5.1 核心用户流程

```text
用户打开桌面客户端
  -> 输入抖音视频 URL
  -> 点击确认
  -> 客户端切换到处理视图并展示下载进度
  -> 自动提取音频
  -> 自动调用 Qwen3-ASR-0.6B 转写
  -> 自动调用 InsightFlow 生成启发话题点
  -> 在界面中展示话题点和文字稿
  -> 用户复制文本或导出文件
```

### 5.2 功能需求

- URL 输入框：支持粘贴单个抖音视频链接。
- 确认按钮：触发下载、校验、音频提取、ASR 转写和 InsightFlow 话题点生成。
- 进度展示：至少包含 `等待输入`、`视频提取中`、`视频转译中`、`话题点生成中`、`文字稿完成`、`失败`。
- 话题点结果区：展示 InsightFlow 生成的开放式启发话题点。
- 文本结果区：展示完整转写文本。
- 导出功能：支持导出 `txt` 和 `md`。
- 历史记录：可选，保存最近处理过的视频 URL、标题、输出文件路径和处理时间。
- 模型管理：可选，展示模型是否已下载、模型路径、重新下载按钮。

### 5.3 非功能需求

- 本地优先：视频和音频默认只在本地处理；如果 InsightFlow 配置云端 LLM，文字稿会发送到对应 LLM API，需要在设置中明确提示。
- 可恢复：下载或识别失败时保留错误日志和可重试入口。
- 可解释：用户能看到当前处理阶段，而不是长时间无反馈。
- 资源可控：ASR worker 独立进程运行，客户端可以取消任务并释放资源。
- 可扩展：后续支持批量 URL、字幕导出、OCR 画面文字识别。

## 6. 桌面客户端 UI 与交互设计

### 6.1 设计目标

桌面客户端采用单窗口、单任务流设计。用户不需要理解 `yt-dlp`、`ffmpeg`、ASR 模型或 InsightFlow，只需要完成一个动作：粘贴视频 URL 并点击确认。系统随后按阶段展示进度，最终给出可直接阅读的文字稿和可继续追问的启发话题点。

界面风格建议：

- 工具型桌面应用，保持安静、清晰、可扫描。
- 首屏直接展示 URL 输入，不显示进度条和文字稿结果区。
- 用户点击确认后，界面切换到处理视图，URL 输入区域不再显示。
- 重点状态只保留三类：输入、处理中、已完成。
- 进度信息使用步骤条和进度条组合，避免用户误判程序卡住。

### 6.8 交互状态表

| 状态 | 触发条件 | UI 表现 | 用户可操作 |
|------|----------|---------|------------|
| 等待输入 | 应用启动或处理新 URL | URL 输入框为空，确认按钮不可用 | 粘贴 URL |
| 可提交 | URL 格式合法 | 确认按钮可用 | 点击确认 |
| 视频提取中 | 用户点击确认 | 切换到处理视图，隐藏 URL 输入区，展示下载/提取进度 | 可取消任务 |
| 视频转译中 | 音频提取成功 | 展示 ASR 转译进度，提示使用 Qwen3-ASR-0.6B | 可取消任务 |
| 话题点生成中 | ASR 输出文本成功 | 展示文字稿，同时展示 InsightFlow 生成话题点进度 | 可取消任务 |
| 完成 | InsightFlow 输出话题点成功 | 展示结果总览卡片和导出按钮 | 点击结果卡片、复制、导出、处理新 URL |
| 部分完成 | ASR 成功但 InsightFlow 失败 | 展示文字稿结果卡片，并提示话题点生成失败 | 点击文字稿卡片、重试生成话题点、导出文字稿 |
| 失败 | 下载、提取或转译失败 | 展示错误原因和重试入口 | 重试、修改 URL |

### 6.9 文案与组件约束

- 主按钮文案使用 `确认并转文字`。
- 进度阶段文案固定为 `视频提取中`、`视频转译中`、`话题点生成中`、`文字稿完成`。
- URL 输入区域只在 `等待输入` 和 `可提交` 状态显示，处理中和完成态不显示。
- 完成态主界面只展示结果总览卡片，不直接铺满完整内容。
- 点击 `启发话题点` 卡片打开结果详情浮窗，并默认选中 `启发话题点` tab。
- 点击 `完整文字稿` 卡片打开结果详情浮窗，并默认选中 `完整文字稿` tab。
- `部分完成` 状态只展示 `完整文字稿` 卡片，`启发话题点` 卡片显示失败态和重试入口。
- 结果详情浮窗内部内容区域独立滚动，浮窗外的主界面不滚动。
- 浮窗支持 `Esc` 关闭，关闭后回到结果总览页。
- 导出按钮在文字稿生成前置灰；文字稿生成后启用文字稿导出，话题点生成后启用完整 Markdown 导出。
- 取消任务只在处理中显示，避免完成态出现无意义操作。

## 7. 端到端处理流程

### 7.1 总体流程

```text
输入 URL
  -> yt-dlp 下载视频
  -> ffprobe 校验媒体文件
  -> ffmpeg 提取 16 kHz 单声道 WAV
  -> Qwen3-ASR-0.6B 转写音频
  -> 文本清洗与分段
  -> 保存 transcript.txt / transcript.md
  -> InsightFlow 生成启发话题点
  -> 输出启发话题点和文字稿到客户端结果区
  -> 导出 txt / md / insights.json
```

### 7.2 进程架构

```text
Desktop UI
  -> Tauri Command
  -> Python Worker
      -> yt-dlp
      -> ffprobe
      -> ffmpeg
      -> qwen-asr
      -> embedded InsightFlow module
  -> Result JSON
  -> Desktop UI
```

前端与 worker 建议使用 JSON 作为通信格式。

请求示例：

```json
{

  "language": "Chinese",
  "output_formats": ["txt", "md"],
  "model": "Qwen/Qwen3-ASR-0.6B",
  "generate_insights": true,
  "insightflow_mode": "embedded"
}
```

响应示例：

```json
{
  "status": "completed",
  "video_path": "outputs/7524373044106677544.mp4",
  "audio_path": "work/7524373044106677544.wav",
  "transcript_path": "outputs/7524373044106677544_transcript.txt",
  "insights_path": "outputs/7524373044106677544_insights.json",

  ],
  "text": "这里是转写后的文字内容。"
}
```

### 7.3 ASR 调用示例

```python
import torch
from qwen_asr import Qwen3ASRModel

model = Qwen3ASRModel.from_pretrained(
    "Qwen/Qwen3-ASR-0.6B",
    dtype=torch.bfloat16,
    device_map="cuda:0",
    max_inference_batch_size=4,
    max_new_tokens=4096,
)

results = model.transcribe(
    audio="work/7524373044106677544.wav",
    language="Chinese",
)

text = results[0].text
```

说明：

- 有 NVIDIA GPU 时优先使用 `cuda:0`。
- 无独显或显存不足时，可提供降级模型选项，例如 `SenseVoiceSmall` 或 `faster-whisper small int8`。
- 如果要输出字幕时间戳，可后续接入 `Qwen/Qwen3-ForcedAligner-0.6B`。

### 7.4 InsightFlow 内置调用流程

ASR 完成后，worker 将转写文本保存为 Markdown，再调用复制到本项目内的 InsightFlow 话题点生成模块。`D:\Github\InsightFlow\src\server` 只作为参考代码来源，不参与运行期 import。

推荐调用顺序：

```text
保存 transcript.md
  -> 读取 transcript.md
  -> 调用内置 MarkdownSplitter 切块
  -> 调用内置 QuestionGenerator 生成开放式话题点
  -> 映射为客户端 insights 数据结构
  -> 保存 insights.json / insights.md
```

建议复制和裁剪的参考模块：

```text
llm_knowledge_processing/
  markdown_splitter.py
  question_generator.py
  utils.py
  llm_client.py 或 llm_gateway.py
  config_manager.py 中与 LLM 配置相关的部分
```

内置模块建议输出统一结构：

```json
{
  "file_id": "transcript-file-id",
  "insights": [
    {
      "id": 1,
      "text": "为什么流程编排可能比单点模型能力更关键？",
      "label": "",
      "chunk_id": 12
    }
  ]
}
```

如果开发时为了复用原 FastAPI 路由，也可以把相关路由复制到本项目内并由 worker 启动一个随应用生命周期管理的本地服务。但该服务必须来自本项目打包内容，不能依赖 `D:\Github\InsightFlow\src\server` 原路径。

异常策略：

- 如果内置 InsightFlow 模块初始化失败，客户端进入 `部分完成` 状态，保留文字稿并提示可稍后重试生成话题点。
- 如果 LLM API 配置缺失或调用失败，返回结构化错误并保留文字稿。
- 如果选择本项目内置 FastAPI 服务封装，worker 需要管理服务启动、健康检查、超时和关闭。

## 10. 异常处理策略

### 10.1 页面需要登录或风控拦截

可能表现：

- 下载失败。
- 返回登录页。
- 无法提取视频信息。
- 只下载到异常 HTML 内容。

处理方式：

```powershell
yt-dlp --cookies-from-browser chrome `
  -o "outputs/%(id)s.%(ext)s" `
  "https://www.douyin.com/video/7524373044106677544"
```

说明：

- `--cookies-from-browser chrome` 可复用本机 Chrome 登录态。
- 仅适用于用户对该内容有访问权限的情况。

### 10.2 下载格式不符合预期

可先查看可用格式：

```powershell
yt-dlp -F "https://www.douyin.com/video/7524373044106677544"
```

再指定格式下载：

```powershell
yt-dlp -f "best" `
  -o "outputs/%(id)s.%(ext)s" `
  "https://www.douyin.com/video/7524373044106677544"
```

### 10.3 HEVC 兼容性问题

本次下载的视频编码为 `hevc`。部分旧播放器或浏览器可能无法直接播放。

如需更高兼容性，可转码为 H.264：

```powershell
ffmpeg -i "outputs/7524373044106677544.mp4" `
  -c:v libx264 -c:a aac `
  "outputs/7524373044106677544_h264.mp4"
```

### 10.4 ASR 模型不可用

可能表现：

- 模型权重未下载。
- Python 环境缺少 `qwen-asr`、`torch` 或 CUDA 依赖。
- 显存不足导致识别失败。
- 首次下载模型耗时过长，用户误以为程序卡死。

处理方式：

- 首次运行时单独做模型初始化检测。
- 在 UI 中展示模型下载进度和本地缓存路径。
- worker 返回结构化错误码，例如 `MODEL_NOT_FOUND`、`CUDA_OUT_OF_MEMORY`、`ASR_RUNTIME_ERROR`。
- 提供降级模型选项，例如 `SenseVoiceSmall`。

### 10.5 InsightFlow 内置模块不可用

可能表现：

- 复制后的 InsightFlow 模块缺少必要文件。
- LLM 配置缺失，例如 `LLM_API_KEY` 未配置。
- Markdown 切块失败。
- 话题点生成返回空列表。
- 如果采用本项目内置 FastAPI 服务封装，本地服务健康检查失败。

处理方式：

- worker 在生成话题点前先做内置模块初始化检查。
- 如果模块不可用，保留已生成的文字稿，不阻断主流程。
- worker 返回结构化错误码，例如 `INSIGHTFLOW_MODULE_MISSING`、`INSIGHTFLOW_CONFIG_MISSING`、`INSIGHTFLOW_LLM_FAILED`、`INSIGHTFLOW_TIMEOUT`。
- 客户端进入 `部分完成` 状态，允许用户稍后重试生成话题点。

## 12. 架构决策记录

### ADR-001：使用 Tauri + React + TypeScript + Python ASR Worker

状态：Accepted

背景：

桌面客户端需要现代 UI、较低资源占用、访问本地文件系统，并能稳定调用 Python 语音识别生态和内置 InsightFlow 话题点生成逻辑。

决策：

使用 `Tauri + React + TypeScript` 构建桌面客户端，使用独立 `Python Worker` 执行下载、转码、ASR 和内置 InsightFlow 话题点生成。

影响：

- 优点：桌面壳轻量，UI 开发体验好，ASR 与 InsightFlow 话题点生成逻辑都与前端解耦。
- 缺点：需要处理 Python 环境、模型权重、worker 打包、复制后的 InsightFlow 模块依赖和进程通信。
- 注意：该技术栈已作为 MVP 固定方案，后续文档不再展开桌面框架备选栈。

### ADR-002：默认 ASR 模型选择 Qwen/Qwen3-ASR-0.6B

状态：Accepted

背景：

需求强调中文识别效果，且允许比 Whisper small / SenseVoiceSmall 消耗更多资源。

决策：

默认使用 `Qwen/Qwen3-ASR-0.6B`，后续保留升级到 `Qwen/Qwen3-ASR-1.7B` 或降级到轻量模型的能力。

影响：

- 优点：中文、粤语和多种中文方言识别能力更适合目标场景。
- 缺点：模型权重和运行依赖较重，首次安装和低配机器体验需要额外设计。
- 扩展：`SenseVoiceSmall` 可作为低资源兜底模型；`Qwen3-ASR-1.7B` 可作为高质量模式。

### ADR-003：复制并内置 InsightFlow 参考实现生成启发话题点

状态：Accepted

背景：

需求从“视频转文字”扩展为“从文字稿生成具有启发的话题点”。本地已有 InsightFlow 服务端参考代码，具备 Markdown 文本切块、LLM 生成开放式问题和结果组织能力。

决策：

以 `D:\Github\InsightFlow\src\server` 作为参考代码来源，将必要模块复制、裁剪并内置到本项目 `worker/insightflow/`。桌面客户端在 ASR 完成后，将文字稿保存为 Markdown，通过内置模块生成启发话题点。

影响：

- 优点：复用已有问题生成 prompt、LLM 网关和切块能力，减少重复开发，同时避免运行期依赖外部仓库路径。
- 缺点：需要在本项目中维护复制后的 InsightFlow 代码，并跟踪参考仓库后续变更。
- 注意：客户端界面中不使用“问题”作为主名称，而统一命名为“启发话题点”。

## 13. 合规与使用边界

本方案应仅用于以下场景：

- 下载公开视频。
